# Security Policy

## Supported Versions

Security fixes are applied to the latest commit on the `main` branch. Older
commits and forks are not supported.

| Version | Supported |
| --- | --- |
| Latest `main` | Yes |
| Older commits | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
pull requests, or discussions.

Use [GitHub private vulnerability reporting](https://github.com/Sekai-World/sekai-master-api/security/advisories/new)
for this repository. If private reporting is unavailable, contact a repository
maintainer privately through GitHub before disclosing the issue publicly.

Include, where possible:

- A description of the vulnerability and its impact.
- Affected endpoints, versions, or configurations.
- Steps to reproduce or a proof of concept.
- Suggested mitigation, if known.

Do not include production credentials, Bearer tokens, or other secrets in a
report.

## Disclosure Process

Maintainers will assess the report, coordinate a fix, and provide a disclosure
plan when practical. Please allow maintainers reasonable time to investigate
and release a fix before public disclosure.
